## Cross-Phase Integration

**Result:** 46/48 wiring points connected | 4/5 E2E flows complete

### Verified Integration Chains

1. **Phase 2 -> Phase 4 -> Phase 5 render pipeline:** fmt::format_to builds escape strings -> render_to_buffer parses into ScreenBuffer cells -> diff_and_emit produces minimal sequences -> write_stdout sends single POSIX write(). All links live in btop.cpp:733-768.

2. **Phase 3 -> Phase 4 -> Phase 5 data flow:** Linux Proc::collect uses read_proc_file (IODC-01) to populate proc_map (IODC-03) with RingBuffer-backed structs (DATA-02) via enum-indexed arrays (DATA-01). Draw pipeline consumes via CpuField/MemField/NetDir enum access. macOS and FreeBSD implement identical patterns (IODC-04).

3. **Phase 6 sanitizer coverage:** cmake-linux.yml sanitizer job builds full libbtop OBJECT library (includes ALL Phase 2-5 optimized code) with ASan+UBSan/TSan, runs ctest + btop --benchmark to exercise all optimized paths.

4. **Phase 7 benchmark fixes:** bench_proc_collect.cpp sub-benchmarks updated to use Tools::read_proc_file() (Phase 3 optimized path). CI failure detection with ::warning:: annotations replaces silent || true.

5. **Config enum round-trip:** BoolKey/IntKey/StringKey enums with constexpr name tables serialize/deserialize through btop_config.cpp. Zero remaining string-literal accessor calls.
