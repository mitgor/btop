## Cross-Phase Integration

**Result:** 16/18 wiring points connected | 4/5 E2E flows complete

### Verified Integrations

1. **Phase 2 + Phase 3 boundary** -- Phase 3 `proc_map` rebuild produces unchanged `vector<proc_info>&` API. Phase 2 `Fx::uncolor` and `fmt::format_to` in draw pipeline consume this data correctly. Sort step re-establishes deterministic order after hash-map rebuild.

2. **Benchmarks measure optimized code** -- `bench_string_utils.cpp` exercises Phase 2 `Fx::uncolor` parser. `bench_draw.cpp` exercises Phase 2 `fmt::format_to` draw code. `Proc::collect (full)` benchmark exercises Phase 3 hash-map path (when Shared::init() succeeds).

3. **Profiling consistency** -- Phase 3.1 PROFILING.md cross-references Phase 1 nanobench data with 5-point analysis. Results are consistent: Proc::collect dominance confirmed, draw functions lightweight, string utilities no longer hot post-optimization.

4. **CI coverage** -- `benchmark.yml` builds against `libbtop` which includes all Phase 2/3 optimized code. Macro-benchmark exercises full collect+draw pipeline end-to-end.

### Integration Gaps

1. **bench_proc_collect.cpp sub-benchmarks** (PROF-02, IODC-01) -- Sub-benchmarks at lines 44-71 use `std::ifstream` (old pattern) instead of `Tools::read_proc_file()`. They measure the code that Phase 3 replaced, not the replacement.

2. **CI proc benchmark reliability** (PROF-02, PROF-03) -- `.github/workflows/benchmark.yml` runs proc benchmark with `|| true`, silently skipping failures. If `Shared::init()` throws in container environments, zero CI data produced for the Phase 3 I/O path.

### Broken Flow

**Phase 3 POSIX I/O micro-benchmark coverage** -- Breaks at bench_proc_collect.cpp sub-benchmarks which measure old ifstream pattern. CI may silently skip the full Proc::collect benchmark. Macro-benchmark partially compensates.
