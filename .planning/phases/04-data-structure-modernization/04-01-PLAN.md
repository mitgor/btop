Create a `RingBuffer<T>` template class in `src/btop_shared.hpp` (before the namespace declarations, after includes). This replaces `deque<long long>` for all time-series data in Phase 4.

**RingBuffer<T> requirements:**
- Heap-allocated backing store via `std::unique_ptr<T[]>` (NOT stack-allocated -- 11 fields * 4096 entries = 352KB would overflow stack)
- Constructor: `explicit RingBuffer(size_t capacity = 0)` -- allocates backing array once, never reallocates during normal operation
- `push_back(T value)` -- if full, overwrites oldest (implicit pop_front), eliminating the manual `while(size > width*2) pop_front()` loops in collectors
- `pop_front()` -- advance head, decrement size
- `back()` / `front()` -- access newest/oldest elements (both const and non-const overloads)
- `operator[](size_t i)` -- zero-based from oldest (matches deque semantics: `data[0]` = oldest, `data[size()-1]` = newest). Both const and non-const.
- `size()`, `empty()`, `capacity()` -- standard queries
- `clear()` -- reset to empty without deallocating
- `resize(size_t new_capacity)` -- reallocate backing store, preserving the most recent `min(size, new_capacity)` elements. Used on terminal resize.
- **Iterator support**: `begin()` / `end()` returning a random-access iterator (or at minimum a forward iterator). Required because `Graph::_create` uses index-based access via `data.at(i)` patterns, and `std::accumulate` is used in net collectors. Implement as a lightweight wrapper that maps logical index to physical `(head_ + i) % capacity_`.
- Default constructor creates an empty buffer (capacity 0, data_ nullptr). Must be copyable/movable for use in vectors and struct initialization.
- Copy constructor and copy assignment: deep-copy the backing array.
- Move constructor and move assignment: transfer ownership.

**Implementation details:**
```cpp
template<typename T>
class RingBuffer {
    std::unique_ptr<T[]> data_;
    size_t capacity_ = 0;
    size_t head_ = 0;  // index of first (oldest) element
    size_t size_ = 0;
    // ...
};
```

Place the template in `src/btop_shared.hpp` after the `#include` block and `using` declarations but before the `namespace Cpu` block. Add `#include <memory>` and `#include <iterator>` if not already present.
