<objective>
Migrate all collector data structures from unordered_map<string, deque<long long>> to enum-indexed std::array<RingBuffer<long long>, N>, and update all 5 platform collectors plus draw code to use the new types.

Purpose: This is the core performance win of Phase 4. Replacing hash maps with flat arrays eliminates per-lookup string hashing and cache misses. Replacing deques with ring buffers eliminates per-push_back heap allocations and the manual pop_front trimming loops. Together these changes give contiguous-memory O(1) access with zero steady-state allocation.

Output: All time-series data structures migrated. All collectors and draw code updated. Pop_front loops eliminated.
</objective>

RingBuffer<T> template in src/btop_shared.hpp:
```cpp
template<typename T>
class RingBuffer {
public:
    explicit RingBuffer(size_t capacity = 0);
    void push_back(T value);      // overwrites oldest when full
    void pop_front();
    T& back(); T& front();
    T& operator[](size_t i);      // zero-based from oldest
    size_t size() const;
    bool empty() const;
    size_t capacity() const;
    void clear();
    void resize(size_t new_capacity);
    iterator begin(); iterator end();  // forward iteration
    // Copy/move constructors and assignment
};
```

2. **Eliminate all pop_front trimming loops.** The RingBuffer auto-overwrites when full, so:

Instead, ensure ring buffers are initialized with the correct capacity. At the beginning of each collect function where `width` is available and ring buffers might need sizing, add:
```cpp
// Initialize ring buffer capacity if not set (first call or after resize)
auto& buf = cpu.cpu_percent[std::to_underlying(CpuField::total)];
if (buf.capacity() == 0 || buf.capacity() != static_cast<size_t>(width * 2))
    buf.resize(width * 2);
```
Or more efficiently, iterate all fields:
```cpp
for (size_t i = 0; i < static_cast<size_t>(CpuField::COUNT); ++i) {
    if (cpu.cpu_percent[i].capacity() != static_cast<size_t>(width * 2))
```
