**CRITICAL:** The `Mv::to()`, `Mv::r()`, `Mv::l()`, `Mv::u()`, `Mv::d()` functions currently reference `Fx::e` to build dynamic strings. Since `Fx::e` is now `string_view`, these functions must be updated to use the literal `"\x1b["` directly or convert from string_view:

Or simply use the literal: `return "\x1b[" + to_string(line) + ...`. Either approach works since the result is a dynamic string anyway.
