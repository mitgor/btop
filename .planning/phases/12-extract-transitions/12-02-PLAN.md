This task makes two changes to btop.cpp: (A) adds `process_accumulated_state()`, and (B) replaces the main loop internals with event-driven dispatch.

**Part A: Add `process_accumulated_state()` function**

Add this function AFTER the `dispatch_event()` definition (from Plan 01) and BEFORE the signal handler. This function handles multi-step state actions that don't fit into single-event transitions:

```cpp
/// Process accumulated state — executes multi-step actions for non-Running states.
/// Called after event dispatch to handle Reloading, Resizing, Sleeping, Error, Quitting.
/// Returns the resulting AppState after processing.
static AppState process_accumulated_state(AppState state) {
    if (state == AppState::Error) {
        clean_quit(1);
        return state;  // unreachable
    }
    if (state == AppState::Quitting) {
        clean_quit(0);
        return state;  // unreachable
    }
    if (state == AppState::Sleeping) {
        _sleep();
        return AppState::Running;
    }
    if (state == AppState::Reloading) {
        if (Runner::active) Runner::stop();
        Config::unlock();
        init_config(cli.low_color, cli.filter);
        Theme::updateThemes();
        Theme::setTheme();
        Draw::update_reset_colors();
        Draw::banner_gen(0, 0, false, true);
        // Chain to Resizing (reload requires re-layout)
        state = AppState::Resizing;
    }
    if (state == AppState::Resizing) {
        term_resize(true);
        Draw::calcSizes();
        Runner::screen_buffer.resize(Term::width, Term::height);
        Draw::update_clock(true);
        if (Menu::active) Menu::process();
        else Runner::run("all", true, true);
        atomic_wait_for(Runner::active, true, 1000);
        return AppState::Running;
    }
    return state;
}
```

**IMPORTANT notes on process_accumulated_state():**
- The Reloading handler does NOT return -- it falls through to the Resizing check. This preserves the Reloading -> Resizing chain from the current code (line 1367: `Global::app_state.store(AppState::Resizing)`).
- The Sleeping handler calls `_sleep()` directly (unlike the current code which does compare_exchange_strong first). The CAS is no longer needed because we're already managing state through the dispatch return value, not through atomic loads.
- Error and Quitting call `clean_quit()` which terminates the process.
- The Resizing handler calls `term_resize(true)` to force size recalculation (equivalent to current line 1371's `term_resize(state == AppState::Resizing)` when state IS Resizing).

**Part B: Replace main loop internals**

Replace the entire main loop body (inside `while (not true not_eq not false)`, approximately lines 1337-1421) with the event-driven version. The new main loop:

```cpp
//? ------------------------------------------------ MAIN LOOP ----------------------------------------------------
// ... (cli.updates and future_time setup remain unchanged above) ...

try {
    while (not true not_eq not false) {
        auto state = Global::app_state.load(std::memory_order_acquire);

        //? 1. Drain signal events from queue via typed dispatch
        while (auto ev = Global::event_queue.try_pop()) {
            state = dispatch_event(state, *ev);
            Global::app_state.store(state, std::memory_order_release);
            if (state == AppState::Quitting or state == AppState::Error) break;
        }

        //? 2. Process accumulated state (multi-step actions for Reloading/Resizing/Sleeping/Error/Quitting)
        state = process_accumulated_state(state);
        Global::app_state.store(state, std::memory_order_release);

        //? 3. Make sure terminal size hasn't changed (in case of SIGWINCH not working properly)
        term_resize();
        if (Global::app_state.load(std::memory_order_acquire) == AppState::Resizing) {
            state = process_accumulated_state(AppState::Resizing);
            Global::app_state.store(state, std::memory_order_release);
        }

        //? 4. Update clock if needed
        if (state == AppState::Running and Draw::update_clock() and not Menu::active) {
            Runner::run("clock");
        }

        //? 5. Timer tick — dispatch via event system
        if (time_ms() >= future_time and state == AppState::Running) {
            Runner::run("all");
            update_ms = Config::getI(IntKey::update_ms);
            future_time = time_ms() + update_ms;
        }

        //? 6. Input polling — dispatch key input via event system
        for (auto current_time = time_ms(); current_time < future_time and state == AppState::Running; current_time = time_ms()) {

            //? Check for external clock changes and for changes to the update timer
            if (std::cmp_not_equal(update_ms, Config::getI(IntKey::update_ms))) {
                update_ms = Config::getI(IntKey::update_ms);
                future_time = time_ms() + update_ms;
            }
            else if (future_time - current_time > update_ms) {
                future_time = current_time;
            }
            //? Poll for input and dispatch via on_event
            else if (Input::poll(min((uint64_t)1000, future_time - current_time))) {
                if (not Runner::active) Config::unlock();
                auto key = Input::get();
                if (not key.empty()) {
                    if (Menu::active) Menu::process(key);
                    else Input::process(key);
                }
            }

            //? Break the loop at 1000ms intervals or if input polling was interrupted
            else break;
        }
    }
}
```

**Key design decisions for the main loop:**

1. **Signal events use dispatch_event():** The drain loop now calls `dispatch_event()` instead of `std::visit([](const auto& e) { process_signal_event(e); }, *ev)`. This routes through on_event() overloads.

2. **Timer tick stays inline (does NOT go through dispatch_event):** The research (Pitfall 5) warns against restructuring the timer/input coupling. `Runner::run("all")` is called directly, matching current behavior. The on_event(Running, TimerTick) overload exists for testing and future use but is not invoked here.

3. **Key input stays inline (does NOT go through dispatch_event):** The input processing (`Menu::process(key)` / `Input::process(key)`) is kept inline in the main loop. The on_event(Running, KeyInput) overload exists for testing and future use but the direct calls match the current behavior exactly, avoiding any risk of behavioral change. Phase 13's entry/exit actions will formalize this.

4. **process_accumulated_state handles the if/else if chain:** The old `if (state == Error) ... else if (state == Quitting) ... else if (state == Sleeping) ... else if (state == Reloading) ...` chain plus the Resizing handling is now unified in `process_accumulated_state()`.

5. **term_resize() safety check preserved:** The `term_resize()` call on step 3 (without arguments) checks if terminal size changed without SIGWINCH. If it detects Resizing state was set, it chains to process_accumulated_state(Resizing) just like the current code.

6. **Running state guard on input loop:** The inner for-loop now checks `state == AppState::Running` in its condition, matching the current behavior where input polling only happens after all state checks pass (research Pitfall 3).
