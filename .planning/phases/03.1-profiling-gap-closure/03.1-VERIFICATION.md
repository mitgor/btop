### Gaps Summary

One gap was found. It does not block the primary phase goal but represents an incomplete documentation chain.

**Gap: PROFILING.md does not reference the profiling script by name**

The PLAN's `key_links` requires `profile_hotfunctions` to appear in PROFILING.md as evidence that the document records results produced by the script. This pattern is absent. PROFILING.md documents the macOS `sample` command methodology and raw output location (`/tmp/btop_profile.txt`) but does not say "generated by `scripts/profile_hotfunctions.sh`."

**Impact assessment:** The core goal of Phase 3.1 is achieved — ranked hot functions with CPU % exist in PROFILING.md, cross-referenced against nanobench data. The missing script citation is a documentation traceability issue, not a functional gap. A human reading PROFILING.md cannot easily determine how to reproduce the results without knowing to look for `scripts/profile_hotfunctions.sh`.

**Fix:** Add one line to PROFILING.md under "Raw Profile Location":
```
Script used: `bash scripts/profile_hotfunctions.sh 500`
```
This closes the key-link gap and makes the profiling session fully reproducible from the documentation alone.
