```cpp
void process(const std::string_view key) {
    if (key.empty()) return;
    try {
        if (std::holds_alternative<input_state::MenuActive>(fsm_state)) {
            Menu::process(key);
            // If Menu::process() emptied the PDA, transition back to Normal
            // Check via pda.empty() -- but PDA is file-scope in btop_menu.cpp.
            // Instead, check: after Menu::process(), Menu::active will be false
            // if PDA emptied (Menu::process sets it at lines 1828/1838).
            // We read Menu::active here temporarily until Plan 02 removes it.
            if (!Menu::active) {
                exit_menu();
            }
            return;
        }
        // ... rest of existing process logic, using FSM for filtering check
    }
    // ... catch block unchanged
}
```

2. Replace the `auto filtering = Config::getB(BoolKey::proc_filtering)` on line 221 with `auto filtering = std::holds_alternative<input_state::Filtering>(fsm_state);`.

3. Replace the enter-filtering path (lines 345-348 where key is "f" or "/"):
   - Currently: `Config::flip(BoolKey::proc_filtering); Proc::filter = ...; old_filter = ...;`
   - New: Call `enter_filtering()` (which handles all of this).

4. Replace the exit-filtering paths:
   - Lines 308-317 (enter/down accept): Replace `Config::set(StringKey::proc_filter, ...); Config::set(BoolKey::proc_filtering, false); old_filter.clear();` with `exit_filtering(true);`. For the "down" recursive case, call `exit_filtering(true)` BEFORE the recursive `process("down")` so the FSM is in Normal state for the recursive call.
