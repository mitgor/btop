
All tests must pass. Any ASan or UBSan report in the output is a failure.

**Step 3: Brief application exercise under ASan**
```bash
timeout 5 ./build-asan/btop || true
```
Check output for `==ERROR==` ASan/UBSan markers. The `timeout 5` ensures it exits after 5 seconds.

**Step 4: Clean and rebuild TSan build**
```bash
rm -rf build-tsan
cmake -B build-tsan -DBUILD_TESTING=ON -DCMAKE_BUILD_TYPE=Debug -DBTOP_SANITIZER=thread
cmake --build build-tsan -j$(sysctl -n hw.ncpu)
```

**Step 5: Run tests under TSan**
```bash
ctest --test-dir build-tsan --output-on-failure
```
All tests must pass. Any TSan data race report is a failure.

**Step 6: Brief application exercise under TSan**
```bash
timeout 5 ./build-tsan/btop || true
```
Check output for `WARNING: ThreadSanitizer` markers.

**If TSan reports false positives on the EventQueue lock-free pattern:** Create a TSan suppression file at `tests/tsan_suppressions.txt` and set `TSAN_OPTIONS=suppressions=tests/tsan_suppressions.txt` when running. Document any suppressions. This is expected per research pitfall analysis.
