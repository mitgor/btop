<objective>
Migrate the runner thread loop, run(), stop(), on_exit(), and all consumer sites to use the Runner FSM types from Plan 01. Remove the old atomic<bool> flags (active, stopping, waiting). This completes the Runner FSM extraction.

Purpose: Replace implicit flag-based state coordination with explicit RunnerStateVar transitions and RunnerStateTag shadow atomic, making the runner thread's lifecycle states visible and testable.

RUNNER-03 approach: The main-to-runner communication uses three typed mechanisms instead of raw bool flag mutation:
1. **Start commands**: runner_conf struct (typed payload) + binary semaphore (wake signal) — already the existing pattern, now with force_redraw folded in
2. **Stop commands**: RunnerStateTag::Stopping stored to the typed enum atomic (not a raw bool)
3. **Redraw requests**: Runner::request_redraw() method + runner_conf.force_redraw field (replaces Runner::redraw atomic<bool>)

This satisfies RUNNER-03's intent: all main-to-runner communication uses typed enums, typed structs, or typed methods — no raw atomic<bool> flag mutation remains.

Output: Fully migrated runner thread with explicit FSM states. All cross-platform collectors, draw, input, menu, and config code using compatibility wrappers. Dead flags removed.
</objective>
