<objective>
Cache macOS SMC and IOKit connection handles across collection cycles to eliminate repeated open/close overhead.

Purpose: Each btop collection cycle currently constructs and destroys SMCConnection objects (IOServiceGetMatchingServices + IOServiceOpen + IOServiceClose per cycle) and IOHIDEventSystemClient objects. This plan converts these to persistent cached handles with automatic reconnect on stale connections (e.g., after sleep/wake).

Output: Modified SMCConnection as singleton, cached ThermalSensors client, updated callers in btop_collect.cpp, unit tests.

**smc.hpp changes:**

2. Remove `masterPort`, `iterator`, `device` member fields — these are only needed during connection setup, not stored.
3. Add `static SMCConnection& instance()` public method.

7. Modify `getSMCTemp()` to check the return code of `SMCCall()`. If it returns `kIOReturnNotReady`, `kIOReturnAborted`, or `kIOReturnNotResponding`, call `reconnect()` and retry once. This handles sleep/wake invalidation (see commit 1707740 and Pitfall 3 from research).
8. Delete copy constructor and copy assignment: `SMCConnection(const SMCConnection&) = delete; SMCConnection& operator=(const SMCConnection&) = delete;`

**smc.cpp changes:**
1. Implement `SMCConnection::instance()` as `static SMCConnection conn; return conn;`.
2. Refactor constructor to call `connect()`. If `connect()` fails, log warning via Logger but do NOT throw (singleton must not throw on construction — callers check `is_connected()`).
3. Implement `connect()`: extract the current constructor's IOServiceMatching/IOServiceOpen logic. Return bool.
4. Implement `reconnect()`: close existing conn if non-zero, then call connect().
5. Modify `getSMCTemp()`: after `SMCReadKey()`, if result is stale-connection error, call `reconnect()` and retry once:
   ```cpp
   long long SMCConnection::getSMCTemp(char *key) {
       SMCVal_t val;
       kern_return_t result = SMCReadKey(key, &val);
       if (result == kIOReturnNotReady || result == kIOReturnAborted || result == kIOReturnNotResponding) {
           if (reconnect()) result = SMCReadKey(key, &val);
       }
       if (result != kIOReturnSuccess || val.dataSize == 0) return -1;
       // ... existing sp78 decode
   }
   ```
6. Destructor: only `IOServiceClose(conn)` if `conn != 0`.
