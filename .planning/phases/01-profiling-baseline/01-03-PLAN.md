  <action>
Create a Python script that converts nanobench JSON output and btop --benchmark JSON output into the `customSmallerIsBetter` format expected by github-action-benchmark.

**Requirements:**
- Python 3 only (no external dependencies -- CI runners have Python pre-installed)
- Accept one or more input files via CLI arguments
- Each input file is either nanobench JSON or btop benchmark JSON (auto-detect by structure)
- Output the combined `customSmallerIsBetter` JSON array to stdout

**For nanobench JSON:**
- Extract from `results[*].benchmarks[*]`:
  - `name` -> use as benchmark name
  - `median(elapsed)` -> convert from seconds to nanoseconds (multiply by 1e9), use as `value`
  - `unit` -> "ns/op"

**For btop benchmark JSON:**
- Extract from `benchmark.summary`:
  - `mean_wall_us` -> name: "btop_cycle_wall", unit: "us/cycle"
  - `mean_collect_us` -> name: "btop_cycle_collect", unit: "us/cycle"
  - `mean_draw_us` -> name: "btop_cycle_draw", unit: "us/cycle"

**Output format:**
```json
[
  {"name": "Fx::uncolor (colored line)", "unit": "ns/op", "value": 123.4},
  {"name": "Tools::ulen (plain)", "unit": "ns/op", "value": 45.6},
  {"name": "btop_cycle_wall", "unit": "us/cycle", "value": 45000.0}
]
```

**Script structure:**
```python
#!/usr/bin/env python3
"""Convert nanobench and btop benchmark JSON to github-action-benchmark format."""

import json
import sys

def convert_nanobench(data):
    """Convert nanobench JSON to customSmallerIsBetter entries."""
    entries = []
    for result_group in data.get("results", []):
        for bench in result_group.get("benchmarks", []):
            entries.append({
                "name": bench["name"],
                "unit": "ns/op",
                "value": bench["median(elapsed)"] * 1e9
            })
    return entries

def convert_btop_benchmark(data):
    """Convert btop --benchmark JSON to customSmallerIsBetter entries."""
    entries = []
    summary = data.get("benchmark", {}).get("summary", {})
    for key in ["mean_wall_us", "mean_collect_us", "mean_draw_us"]:
        if key in summary:
            short_name = key.replace("mean_", "btop_cycle_").replace("_us", "")
            entries.append({
                "name": short_name,
                "unit": "us/cycle",
                "value": summary[key]
            })
    return entries

def main():
    if len(sys.argv) < 2:
        print("Usage: convert_nanobench_json.py <file1.json> [file2.json ...]", file=sys.stderr)
        sys.exit(1)

    all_entries = []
    for path in sys.argv[1:]:
        with open(path) as f:
            data = json.load(f)
        if "results" in data:
            all_entries.extend(convert_nanobench(data))
        elif "benchmark" in data:
            all_entries.extend(convert_btop_benchmark(data))
        else:
            print(f"Warning: Unknown format in {path}", file=sys.stderr)

    json.dump(all_entries, sys.stdout, indent=2)
    print()  # trailing newline

if __name__ == "__main__":
    main()
```

**Workflow structure:**

```yaml
name: Performance Benchmarks
on:
  push:
    branches: [main]
  pull_request:
    branches: [main]

permissions:
  contents: write       # For updating gh-pages benchmark data
  pull-requests: write  # For posting PR comments on regression

jobs:
  benchmark-linux:
    runs-on: ubuntu-24.04
    steps:
      - uses: actions/checkout@v4

      - name: Install dependencies
        run: |
          sudo apt-get update
          sudo apt-get install -y ninja-build

      - name: Configure
        run: |
          cmake -B build -G Ninja \
            -DCMAKE_BUILD_TYPE=RelWithDebInfo \
            -DCMAKE_CXX_FLAGS="-fno-omit-frame-pointer" \
            -DBTOP_BENCHMARKS=ON

      - name: Build
        run: cmake --build build

      - name: Run micro-benchmarks
        run: |
          ./build/benchmarks/btop_bench_strings --json > bench_strings.json
          ./build/benchmarks/btop_bench_draw --json > bench_draw.json

      - name: Run macro-benchmark
        run: |
          ./build/btop --benchmark 20 > bench_macro.json

      - name: Convert results
        run: |
          python3 benchmarks/convert_nanobench_json.py \
            bench_strings.json bench_draw.json bench_macro.json \
            > benchmark_results.json

      - name: Store benchmark result
        uses: benchmark-action/github-action-benchmark@v1
        with:
          tool: 'customSmallerIsBetter'
          output-file-path: benchmark_results.json
          github-token: ${{ secrets.GITHUB_TOKEN }}
          auto-push: ${{ github.event_name == 'push' && github.ref == 'refs/heads/main' }}
          alert-threshold: '200%'
          comment-on-alert: true
          fail-on-alert: false
          alert-comment-cc-users: '@aristocratos'
          benchmark-data-dir-path: 'dev/bench'

  benchmark-macos:
    runs-on: macos-14
    steps:
      - uses: actions/checkout@v4

      - name: Install dependencies
        run: |
          brew install ninja

      - name: Configure
        run: |
          cmake -B build -G Ninja \
            -DCMAKE_BUILD_TYPE=RelWithDebInfo \
            -DCMAKE_CXX_FLAGS="-fno-omit-frame-pointer" \
            -DBTOP_BENCHMARKS=ON \
            -DCMAKE_CXX_COMPILER=$(brew --prefix llvm)/bin/clang++

      - name: Build
        run: cmake --build build

      - name: Run micro-benchmarks
        run: |
          ./build/benchmarks/btop_bench_strings --json > bench_strings.json
          ./build/benchmarks/btop_bench_draw --json > bench_draw.json

      - name: Run macro-benchmark
        run: |
          ./build/btop --benchmark 20 > bench_macro.json

      - name: Convert results
        run: |
          python3 benchmarks/convert_nanobench_json.py \
            bench_strings.json bench_draw.json bench_macro.json \
            > benchmark_results.json

      # macOS results stored separately, not auto-pushed (Linux is primary baseline)
      - name: Store benchmark result
        uses: benchmark-action/github-action-benchmark@v1
        with:
          tool: 'customSmallerIsBetter'
          output-file-path: benchmark_results.json
          github-token: ${{ secrets.GITHUB_TOKEN }}
          auto-push: false
          alert-threshold: '200%'
          comment-on-alert: true
          fail-on-alert: false
          benchmark-data-dir-path: 'dev/bench-macos'
```

**Key design decisions (from research):**

1. **Alert threshold: 200%** -- Start generous because CI runners have noisy performance. Tighten after baseline stabilizes. This is github-action-benchmark's default and recommended starting point.

2. **fail-on-alert: false** -- Alerts comment on PRs but don't block merge. This prevents false positives from blocking development. Can be changed to `true` once baseline is stable.

3. **Linux is primary, macOS is secondary** -- Only Linux auto-pushes baseline to gh-pages. macOS runs for awareness but doesn't update baseline (avoids baseline conflicts from different hardware).

4. **RelWithDebInfo + -fno-omit-frame-pointer** -- Standard profiling build configuration. Keeps optimizations while enabling stack walking for future profiling runs.

5. **20 benchmark cycles** -- Enough for statistical significance without making CI too slow. Matches the macro-benchmark's auto-warmup from first few cycles.

6. **No FreeBSD benchmark in CI** -- Per research open question #4, FreeBSD VM overhead makes benchmarks too noisy. FreeBSD is build-verified only; performance baseline is manual.

**IMPORTANT:** Follow the exact compiler setup from existing CI workflows. For Linux, the cmake-linux.yml may install specific GCC/Clang versions. For macOS, cmake-macos.yml uses Homebrew LLVM. Match those patterns.
