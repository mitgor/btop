## Objective

Migrate all 235 `Theme::c("string")`, `Theme::g("string")`, and `Theme::dec("string")` call sites to use `ColorKey`/`GradientKey` enum values. Convert the `Meter` and `Graph` classes to store `GradientKey` instead of `string`. Remove the temporary string-accepting overloads. After this plan, zero string-keyed theme lookups remain in the codebase.

## Context

Read these files before starting:

- `src/btop_theme.hpp` — Updated in Plan 01 with ColorKey/GradientKey enums and both enum + string overloads
- `src/btop_draw.hpp` — Meter and Graph class declarations (store `string color_gradient` member)
- `src/btop_draw.cpp` — 183 Theme::c() calls, plus Meter/Graph constructors and rendering code with Theme::g() calls
- `src/btop_menu.cpp` — 48 Theme::c()/g() calls
- `src/btop.cpp` — 4 Theme::c() calls
