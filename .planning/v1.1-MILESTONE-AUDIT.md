## Cross-Phase Integration

**Architecture chain (verified correct):**
```
Phase 10: AppStateTag enum (btop_state.hpp)
    ↓
Phase 11: AppEvent variant + EventQueue (btop_events.hpp)
    ↓
Phase 12: on_event() overloads + dispatch_event() (btop.cpp)
    ↓
Phase 13: AppStateVar variant + transition_to() + on_enter/on_exit (btop.cpp)
    ↓
Main loop: event_queue.try_pop() → dispatch_event() → transition_to()
```

**Legacy infrastructure fully removed:** `state_tag::`, `dispatch_state()`, `process_accumulated_state()`, `process_signal_event()`, all old boolean flag names — zero occurrences in src/.

### Integration Issues

**1. event::ThreadError — dead code path (Medium)**
- `event::ThreadError` is defined in the variant and has an `on_event` handler, but is never pushed to the queue
- Runner thread writes directly to `Global::app_state` atomic on exception, bypassing the event system
- Affects: EVENT-01, RUNNER-03
- Resolution: Phase 14 (RUNNER-03) will wire Runner exceptions through the event queue

**2. SIGTERM not routed (Minor, pre-existing)**
- `event::Quit` comment says "SIGINT / SIGTERM" but SIGTERM has no registered handler
- `kill <pid>` terminates without terminal restoration
- Pre-existing behavior gap, not a regression
