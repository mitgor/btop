<objective>
Route all runner thread error paths through the event-driven FSM architecture and remove write-only runner_var dead code.

Purpose: The runner thread currently bypasses the event system on errors by directly writing `Global::app_state.store(AppStateTag::Error)`. This creates shadow atomic writes that circumvent `transition_to()`, leaving `app_var` (the authoritative variant) and `Global::app_state` (the shadow atomic) out of sync. This plan makes error paths pure by pushing `event::ThreadError` into the event queue, letting the main loop dispatch and transition properly. It also removes `runner_var` which is write-only dead code.

Output: Modified `src/btop.cpp`, `src/btop_state.hpp`, updated tests, removed dead runner variant infrastructure.
</objective>

  <action>
  **Goal:** Replace direct shadow atomic writes on runner error paths with event queue pushes, making on_event(Running, ThreadError) reachable.

  **Step 1 — Update on_event(Running, ThreadError) in src/btop.cpp (around line 333):**

  Change:
  ```cpp
  static AppStateVar on_event(const state::Running&, const event::ThreadError&) {
      return state::Error{"Thread error"};
  }
  ```
  To:
  ```cpp
  static AppStateVar on_event(const state::Running&, const event::ThreadError&) {
      return state::Error{std::move(Global::exit_error_msg)};
  }
  ```
  Rationale: The runner thread writes `Global::exit_error_msg` before pushing the event. The event_queue push uses release ordering on head_, and the main thread's try_pop uses acquire ordering, so the message is guaranteed visible when this handler runs. Moving from exit_error_msg avoids a copy and clears the source.

  **Step 2 — Add on_exit(Running, Error) handler near line 987:**

  Add a new on_exit overload for Running -> Error that stops the runner:
  ```cpp
  static void on_exit(const state::Running&, const state::Error&) {
      if (Runner::is_active()) Global::runner_state_tag.store(Global::RunnerStateTag::Stopping, std::memory_order_release);
  }
  ```
  This replaces the `runner_state_tag.store(Stopping)` that was previously done directly in the runner catch block.

  **Step 3 — Replace runner exception handler (around line 717-722):**

  Change:
  ```cpp
  catch (const std::exception& e) {
      Global::exit_error_msg = fmt::format("Exception in runner thread -> {}", e.what());
      Global::app_state.store(AppStateTag::Error, std::memory_order_release);
      Input::interrupt();
      Global::runner_state_tag.store(Global::RunnerStateTag::Stopping, std::memory_order_release);
  }
  ```
  To:
  ```cpp
  catch (const std::exception& e) {
      Global::exit_error_msg = fmt::format("Exception in runner thread -> {}", e.what());
      Global::event_queue.push(event::ThreadError{});
      Input::interrupt();
  }
  ```
  Key changes: (1) Removed direct `app_state.store(Error)` — now flows through event queue. (2) Removed `runner_state_tag.store(Stopping)` — now done by on_exit(Running, Error). (3) Kept `Input::interrupt()` to wake the main loop from pselect(). (4) Kept `exit_error_msg` write as sideband for the error message (trivially_copyable constraint on AppEvent prevents putting std::string in the event).

  **Step 4 — Replace runner safety-check error (around line 539-543):**

  Change:

  To:
  ```cpp
  if (Runner::is_active()) {
      Global::exit_error_msg = "Runner thread failed to get active lock!";
      Global::event_queue.push(event::ThreadError{});
      Input::interrupt();
  }
  ```
  Same pattern: write message to sideband, push event, interrupt main loop. The on_exit(Running, Error) handler stops the runner.
