Key implementation details:
- Use `os.wait4()` for RSS (NOT subprocess + parsing /usr/bin/time). The research confirms this is more reliable.
- On macOS, `ru_maxrss` returns BYTES. Divide by 1024*1024 for MB.
- Use `statistics.mean/median/stdev` from stdlib (no external deps).
- Record binary sizes via `os.path.getsize()`.
- Include `get_system_info()` capturing uname, sw_vers, clang++ --version, sysctl cpu/cores.
- Do NOT use powermetrics (requires sudo, not reproducible).
- Script must be runnable with just `python3 scripts/measure_performance.py` with no arguments.
- If baseline binary path doesn't exist, print clear error with instructions rather than crashing.

Model the script on the existing `benchmark_comparison.py` but with these improvements:
- Build from source (don't hardcode binary paths as constants used directly -- derive from build)
- Use v12_ prefix for output files to distinguish from v1.0 results
- Add /usr/bin/time cross-validation
- Include methodology comments in the script itself
