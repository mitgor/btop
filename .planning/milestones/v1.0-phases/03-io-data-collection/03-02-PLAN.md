<objective>
Replace O(N) linear PID scans with O(1) hash-based lookups in process collectors across all three platforms (Linux, macOS, FreeBSD).

Purpose: With 500 running processes, the current `rng::find(current_procs, pid)` performs ~250,000 comparisons per update cycle (500 PIDs x 500-element vector). The dead-process cleanup `v_contains(found, element.pid)` adds another O(N^2) pass. Converting to `unordered_map<size_t, proc_info>` for the lookup map and `unordered_set<size_t>` for alive-PID tracking reduces both to O(N) total.

Output: All three platform collectors use hash-map PID lookup with vector view for sorting/return.
</objective>
