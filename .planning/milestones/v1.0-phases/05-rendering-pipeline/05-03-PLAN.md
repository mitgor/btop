<objective>
Implement differential terminal output using a double-buffered cell buffer. Draw function output strings are parsed into a screen-wide cell buffer, diffed against the previous frame, and only changed cells emit escape sequences. This is the incremental migration approach (strategy (a) from research) -- draw functions continue producing escape-sequence strings, but the output goes through the cell buffer for diffing.

Purpose: During steady-state operation (no resize, no config change), the vast majority of terminal cells are identical between frames -- only the rightmost graph columns, numeric values (percentages, temperatures, speeds), and clock updates change. Emitting escape sequences for only the ~5-15% of cells that actually changed reduces bytes written per frame by 30-50%+, directly reducing CPU time spent on terminal I/O.

Output: Cell struct + ScreenBuffer class with double-buffering and diff, escape-string-to-buffer parser, Runner thread integration, unit tests.

**Parser tradeoff acknowledgment:** This plan ships the escape-string-to-cell-buffer parser (strategy (a) from research) as the production implementation rather than migrating draw functions to direct cell writes (strategy (b)). This is an intentional incremental approach. The parser adds per-frame CPU overhead, but the differential output mechanism saves far more bytes than the parser costs: the 30-50% byte-reduction target is achievable because the diff operates on O(width*height) cells and skips the vast majority during steady state, while the parser cost is proportional to the draw output string length which is already being generated. The net bytes-written reduction is positive because the savings from emitting only ~5-15% of cells outweigh the cost of parsing the full draw output in memory. Direct cell writes for hot-path draw functions (particularly Graph output) can be pursued in a future optimization phase if profiling indicates the parser is a measurable bottleneck.
</objective>

    //? Cell represents a single terminal cell with character, colors, and attributes
    struct Cell {
        char ch[4] = {' ', 0, 0, 0};  // UTF-8 encoded character, null-padded
        uint8_t ch_len = 1;            // Length of UTF-8 character in bytes

        uint32_t fg = 0;  // Foreground: 0=default, bit 24 set=truecolor (RGB in low 24 bits), else 256-color index
        uint32_t bg = 0;  // Background: same encoding

        uint8_t attrs = 0;  // Bitfield: bit 0=bold, 1=dim, 2=italic, 3=underline, 4=blink, 5=strikethrough

        bool operator==(const Cell& o) const {
            return fg == o.fg && bg == o.bg && attrs == o.attrs
                && ch_len == o.ch_len && std::memcmp(ch, o.ch, ch_len) == 0;
        }
        bool operator!=(const Cell& o) const { return !(*this == o); }

        void set_char(char c) { ch[0] = c; ch[1] = 0; ch_len = 1; }
        void set_char(const char* utf8, uint8_t len) {
            std::memcpy(ch, utf8, len);
            if (len < 4) ch[len] = 0;
            ch_len = len;
        }
    };

    //? Double-buffered screen cell buffer for differential rendering
    class ScreenBuffer {
        std::vector<Cell> cells_a, cells_b;  // Double buffer
        Cell* current = nullptr;   // Points to active buffer
        Cell* previous = nullptr;  // Points to previous frame buffer
        int w = 0, h = 0;
        bool force_full = true;    // Force full redraw (first frame, resize)

    public:
        void resize(int width, int height);
        void swap();  // Swap current/previous buffers
        void set_force_full() { force_full = true; }
        bool needs_full() const { return force_full; }
        void clear_force_full() { force_full = false; }

        Cell& at(int x, int y) { return current[y * w + x]; }
        const Cell& at(int x, int y) const { return current[y * w + x]; }
        const Cell& prev_at(int x, int y) const { return previous[y * w + x]; }
        int width() const { return w; }
        int height() const { return h; }
        void clear_current();  // Reset all current cells to default
    };

    //? Parse an escape-sequence string and render it into the ScreenBuffer

    //? Diff current vs previous buffer and emit minimal escape sequences

        w = width; h = height;

        cells_a.resize(sz);

    **Step 3: Implement render_to_buffer() in btop_draw.cpp:**

    This is the escape-string-to-cell-buffer parser. It processes the concatenated draw output (which contains Mv::to cursor moves, SGR color/attribute codes, and UTF-8 characters) and populates the cell buffer.

    Key CSI sequences to handle:
    - **CUP** (Cursor Position): `\x1b[Y;Xf` or `\x1b[Y;XH` -- set cx, cy
    - **SGR** (Select Graphic Rendition): `\x1b[Nm` or `\x1b[N;N;...m` -- set fg, bg, attrs
      - 0 = reset, 1 = bold, 2 = dim, 3 = italic, 4 = underline, 5 = blink, 9 = strikethrough
      - 22 = unbold/undim, 23 = un-italic, 24 = un-underline, 25 = un-blink, 29 = un-strike
      - 38;2;R;G;B = truecolor fg, 48;2;R;G;B = truecolor bg
      - 38;5;N = 256-color fg, 48;5;N = 256-color bg
      - 39 = default fg, 49 = default bg
    - **CUU/CUD/CUF/CUB** (Relative moves): `\x1b[NA/B/C/D` -- adjust cx, cy
    - **Cursor right** (Mv::r): `\x1b[NC` -- cx += N

    Implementation outline:
    ```cpp
    void Draw::render_to_buffer(ScreenBuffer& buf, const std::string& s) {
        int cx = 0, cy = 0;
        uint32_t fg = 0, bg = 0;
        uint8_t attrs = 0;

        size_t i = 0;
        const size_t len = s.size();
        while (i < len) {
            if (s[i] == '\x1b' && i + 1 < len && s[i + 1] == '[') {
                // Parse CSI sequence
                i += 2;
                // Collect parameters (semicolon-separated numbers)
                // Determine final character (m, f, H, A, B, C, D, etc.)
                // Apply to cursor position or color state
                // ... (detailed CSI parser)
            }
            else if (s[i] == '\n') {
                cy++; cx = 0; i++;
            }
            else {
                // Regular character -- write to cell buffer
                if (cx >= 0 && cx < buf.width() && cy >= 0 && cy < buf.height()) {
                    auto& cell = buf.at(cx, cy);
                    // Decode UTF-8 character length
                    uint8_t char_len = 1;
                    if ((s[i] & 0x80) == 0) char_len = 1;
                    else if ((s[i] & 0xE0) == 0xC0) char_len = 2;
                    else if ((s[i] & 0xF0) == 0xE0) char_len = 3;
                    else if ((s[i] & 0xF8) == 0xF0) char_len = 4;

                    cell.set_char(s.data() + i, char_len);
                    cell.fg = fg;
                    cell.bg = bg;
                    cell.attrs = attrs;
                    cx++;
                }
                // Advance past UTF-8 character
                if ((s[i] & 0x80) == 0) i += 1;
                else if ((s[i] & 0xE0) == 0xC0) i += 2;
                else if ((s[i] & 0xF0) == 0xE0) i += 3;
                else if ((s[i] & 0xF8) == 0xF0) i += 4;
                else i += 1;  // Invalid UTF-8, skip byte
            }
        }
    }
    ```

    **Critical CSI parser details:**
    - Parameters are parsed as a vector of ints separated by ';'
    - Final byte determines the command (m=SGR, f/H=CUP, A=up, B=down, C=right, D=left)
    - For SGR (m): iterate params and update fg/bg/attrs. For 38;2;R;G;B, consume 5 params. For 38;5;N, consume 3 params.
    - For truecolor storage: `fg = (1u << 24) | (R << 16) | (G << 8) | B` (high bit marks truecolor)
    - For 256-color storage: `fg = N + 1` (add 1 to distinguish from 0=default)

    **Step 4: Implement diff_and_emit() in btop_draw.cpp:**

    ```cpp
    void Draw::diff_and_emit(const ScreenBuffer& buf, std::string& out) {
        out.clear();
        out.reserve(static_cast<size_t>(buf.width()) * buf.height() * 4);

        int last_x = -1, last_y = -1;
        uint32_t last_fg = UINT32_MAX, last_bg = UINT32_MAX;
        uint8_t last_attrs = UINT8_MAX;

        for (int y = 0; y < buf.height(); ++y) {
            for (int x = 0; x < buf.width(); ++x) {
                const auto& c = buf.at(x, y);
                const auto& p = buf.prev_at(x, y);

                if (c == p) continue;  // Skip unchanged cells

                // Emit cursor move if not sequential
                if (y != last_y || x != last_x + 1) {
                    fmt::format_to(std::back_inserter(out), "\x1b[{};{}f", y + 1, x + 1);
                }

                // Emit attribute changes
                if (c.attrs != last_attrs) {
                    emit_attrs(c.attrs, last_attrs, out);  // Helper function
                    last_attrs = c.attrs;
                }

                // Emit fg color change
                if (c.fg != last_fg) {
                    emit_color(c.fg, true, out);  // true = foreground
                    last_fg = c.fg;
                }

                // Emit bg color change
                if (c.bg != last_bg) {
                    emit_color(c.bg, false, out);  // false = background
                    last_bg = c.bg;
                }

                // Emit character
                out.append(c.ch, c.ch_len);

                last_x = x;
                last_y = y;
            }
        }
    }
    ```

    Implement helper functions:
    - `emit_attrs(uint8_t new_attrs, uint8_t old_attrs, string& out)`: Emit SGR codes for changed attributes
    - `emit_color(uint32_t color, bool is_fg, string& out)`: Emit color escape sequence from packed representation
