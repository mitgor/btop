<objective>
Add a `last_data_back` early-return optimization and `out.reserve()` to the Graph class so that when the newest data point is unchanged, the entire erase-shift-create-rebuild cycle is skipped.

Purpose: During steady-state operation, graphs receive one new data point per frame. When the newest value equals the previous frame's value (common for idle metrics), Graph::operator() currently still erases, shifts, recomputes, and rebuilds. The `last_data_back` check skips this entirely, returning the cached `out` string. The `out.reserve()` in `_create()` eliminates per-frame reallocation for the cases where a full rebuild is needed. Per-column caching was considered but rejected due to complexity of parsing color-interleaved row strings; the `last_data_back` approach achieves the majority of the benefit with minimal risk. When combined with the cell buffer (Plan 03), the diff mechanism provides the remaining savings.

Output: Modified Graph class with last_data_back optimization and out.reserve(), unit tests confirming output identity, benchmark quantifying improvement.
</objective>

    **Design note:** Per-column symbol caching (`cached_columns`) was considered but rejected. The Graph implementation interleaves color escape codes with symbol characters in row strings, making column-level extraction complex and fragile. The `last_data_back` approach below achieves 80% of the benefit with 20% of the complexity. The cell buffer (Plan 03) provides the remaining optimization by diffing at the screen level.

    **Step 1: Add out.reserve() in _create() (btop_draw.cpp):**

    In `Graph::_create()` (the `out` rebuild section, lines 446-461):
    1. Before `out.clear()`, call `out.reserve()` with a size estimate: `width * height * 8` (accounts for UTF-8 chars + color codes + cursor moves). This prevents reallocation on every frame.
    2. The existing rebuild logic remains -- it is already O(height) string concatenations, which is fast for typical graph heights (1-10 rows).

    In `Graph::operator()` (lines 493-508):
    1. The existing `data_same` early return is the primary cache (already in place).
    2. Add a `last_data_value` member to track the previous rightmost data point. If `data.back() == last_data_value`, we can skip the full _create + rebuild since the output would be identical. This is a **stronger** cache than `data_same` which is controlled by the caller.

    **Implementation:**

    a) Add to Graph private members in btop_draw.hpp:
    ```cpp
    long long last_data_back = LLONG_MIN;  // Track last data.back() value for skip optimization
    ```

    b) In `Graph::operator()` (btop_draw.cpp), after the `data_same` check, add:
    ```cpp
    // If the newest data point hasn't changed, the graph output is identical
    // (shift produces same visual result when rightmost value is unchanged)
    long long current_back = data.back();
    if (current_back == last_data_back && !out.empty()) {
        return out;
    }
    last_data_back = current_back;
    ```
    Place this AFTER the `if (data_same) return out;` line and BEFORE the `current = not current` toggle. This way, if the actual data value at the tip hasn't changed, we skip the expensive erase-shift-create-rebuild cycle entirely.

    c) In `Graph::_create()`, before the `out.clear()` line (around line 446), add:
    ```cpp
    out.reserve(width * height * 8);  // Pre-allocate for escape codes + UTF-8 chars + cursor moves
    ```

    d) In the `Graph::Graph(...)` constructor, initialize `last_data_back` at the end:
    ```cpp
    if (data.size() > 0) {
        last_data_back = data.back();
    }
    ```

    e) Reset `last_data_back = LLONG_MIN` if the graph is reconstructed (already handled by constructor re-initialization since Graph objects are reconstructed on resize).

    **Why this approach instead of per-column caching:**
    The research suggested per-column caching, but the actual Graph implementation interleaves color escape codes with symbol characters in the row strings, making column-level extraction complex and fragile. The `last_data_back` check achieves 80% of the benefit (skipping unchanged frames) with 20% of the complexity. The `out.reserve()` handles the remaining allocation overhead. The cell buffer (Plan 03) will provide the remaining optimization by diffing at the screen level.
