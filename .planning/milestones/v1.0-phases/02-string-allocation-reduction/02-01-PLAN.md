<objective>
Replace the std::regex-based Fx::uncolor() with a hand-written single-pass ANSI escape stripper that eliminates regex allocation overhead, and change five string utility functions from pass-by-value to pass-by-const-reference to eliminate unnecessary copies at 127+ call sites per frame.

Purpose: These are the two highest-impact, lowest-risk string allocation reductions identified in profiling. The uncolor regex is a known allocation spike (std::regex allocates internally on every call), and the by-value parameters force a copy on every function entry even when the string needs no modification.

Output: Modified btop_tools.hpp/cpp with zero-alloc uncolor and copy-free function signatures, plus a comprehensive uncolor correctness test suite.
</objective>

```cpp
namespace Fx {
    string uncolor(const string& s) {
        string out;
        out.reserve(s.size());
        for (size_t i = 0; i < s.size(); ) {
            if (s[i] == '\033' && i + 1 < s.size() && s[i + 1] == '[') {
                // Potential SGR sequence: ESC [ <digits-and-semicolons> m
                size_t j = i + 2;
                while (j < s.size() && ((s[j] >= '0' && s[j] <= '9') || s[j] == ';')) {
                    j++;
                }
                if (j < s.size() && s[j] == 'm') {
                    // Valid SGR sequence — skip entirely
                    i = j + 1;
                    continue;
                }
                // Not terminated by 'm' — not a color sequence.
                // Copy the ESC character and continue (matches regex behavior:
                // regex wouldn't match this sequence either, but the ESC[ prefix
                // characters are already consumed positionally).
            }
            out += s[i++];
        }
        return out;
    }
}
```

Key correctness requirements:
- Must handle: `\033[0m`, `\033[1m`, `\033[38;5;196m`, `\033[38;2;255;128;0m`, `\033[48;2;255;128;0m`
- Must NOT strip non-SGR sequences (e.g., `\033[10C` cursor movement — these end in 'C' not 'm')
- Must handle empty strings, strings with no escapes, strings that are only escapes, consecutive escapes, partial/malformed sequences
- Must use forward-copy pattern (O(n)), NOT erase-in-loop (O(n^2)) — the old commented-out implementation used erase and caused performance issues on musl

string luresize(const string& str, const size_t len, bool wide) {
    if (len < 1 or str.empty())
        return "";

    for (size_t x = 0, last_pos = 0, i = str.size() - 1; i > 0 ; i--) {
        if (wide and static_cast<unsigned char>(str.at(i)) > 0xef) {
            x += 2;
