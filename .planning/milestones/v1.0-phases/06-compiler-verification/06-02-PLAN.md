  <action>
    Execute sanitizer sweeps, PGO build, and mimalloc evaluation on the local machine (macOS). Document all results in `06-RESULTS.md`. Work through these steps in order, capturing output at each stage.

    **Step 1 -- ASan+UBSan sweep:**
    ```bash
    cmake -B build-asan -G Ninja \
      -DCMAKE_BUILD_TYPE=Debug \
      -DBTOP_SANITIZER="address,undefined" \
      -DBUILD_TESTING=ON
    cmake --build build-asan
    ```
    Then run tests and benchmark:
    ```bash
    ASAN_OPTIONS=detect_leaks=0:halt_on_error=1 \
    UBSAN_OPTIONS=halt_on_error=1:print_stacktrace=1 \
    ctest --test-dir build-asan --output-on-failure
    ```
    ```bash
    ASAN_OPTIONS=detect_leaks=0:halt_on_error=1 \
    UBSAN_OPTIONS=halt_on_error=1:print_stacktrace=1 \
    ./build-asan/btop --benchmark 20
    ```
    Note: Use `detect_leaks=0` on macOS where leak detection is not supported by default. On Linux CI, Plan 01's workflow uses `detect_leaks=1`.

    If sanitizer errors appear:
    - If they are pre-existing issues (not from our optimization phases), document them in RESULTS.md under "Pre-existing Issues" and note they are not regressions.
    - If they are true regressions from Phases 2-5, fix the underlying code and re-run. Document what was found and fixed.
    - If TSan reports false positives from btop's custom `atomic_lock` pattern, document them and create a `tsan.supp` suppression file if needed.

    **Step 2 -- TSan sweep:**
    ```bash
    cmake -B build-tsan -G Ninja \
      -DCMAKE_BUILD_TYPE=Debug \
      -DBTOP_SANITIZER="thread" \
      -DBUILD_TESTING=ON
    cmake --build build-tsan
    ```
    Then run tests and benchmark:
    ```bash
    TSAN_OPTIONS=halt_on_error=1:second_deadlock_stack=1 \
    ctest --test-dir build-tsan --output-on-failure
    ```
    ```bash
    TSAN_OPTIONS=halt_on_error=1:second_deadlock_stack=1 \
    ./build-tsan/btop --benchmark 20
    ```
    TSan has 5-15x overhead; 20 benchmark cycles is sufficient for concurrency verification.

    **Step 3 -- PGO build and measurement:**

    First check if `hyperfine` is installed (`which hyperfine`). If not, install via `brew install hyperfine`.

    Run the PGO build script:
    ```bash
    CXX=$(which clang++) ./scripts/pgo-build.sh Release
    ```

    Then run hyperfine comparison:
    ```bash
    hyperfine \
      --warmup 3 \
      --min-runs 10 \
      --command-name 'without-pgo' './build-pgo-gen/btop --benchmark 50' \
      --command-name 'with-pgo' './build-pgo-use/btop --benchmark 50' \
      --export-json pgo_benchmark.json
    ```

    Also build a baseline Release build WITHOUT PGO for fair comparison (the PGO-gen build has instrumentation overhead):
    ```bash
    cmake -B build-release -G Ninja \
      -DCMAKE_BUILD_TYPE=Release \
      -DCMAKE_CXX_COMPILER=$(which clang++) \
      -DBTOP_LTO=ON
    cmake --build build-release

    hyperfine \
      --warmup 3 \
      --min-runs 10 \
      --command-name 'release-no-pgo' './build-release/btop --benchmark 50' \
      --command-name 'release-with-pgo' './build-pgo-use/btop --benchmark 50' \
      --export-json pgo_vs_release.json
    ```

    **Step 4 -- mimalloc evaluation:**

    Check if mimalloc is installed (`brew list mimalloc` or `brew install mimalloc`).

    Run hyperfine comparison with mimalloc via DYLD_INSERT_LIBRARIES:
    ```bash
    MIMALLOC_LIB="$(brew --prefix mimalloc)/lib/libmimalloc.dylib"

    hyperfine \
      --warmup 3 \
      --min-runs 10 \
      --command-name 'default-allocator' './build-release/btop --benchmark 50' \
      --command-name 'mimalloc' "DYLD_INSERT_LIBRARIES=$MIMALLOC_LIB ./build-release/btop --benchmark 50" \
      --export-json mimalloc_benchmark.json
    ```

    Note: If DYLD_INSERT_LIBRARIES is blocked by SIP on macOS, document that limitation and provide the equivalent Linux LD_PRELOAD command for CI/Linux testing. A "could not preload" or silent failure is a valid documented result.
