1. **Modify src/btop.cpp:** Add benchmark mode execution path in `btop_main()`. The key insight is that benchmark mode must initialize enough state for collectors and draw functions to work, but must NOT initialize the terminal (no raw mode, no alternate screen, no signal handlers for terminal resize).

   **Add after CLI parsing succeeds and before the normal terminal initialization path:**

```cpp
if (cli.benchmark_cycles.has_value()) {
    // Benchmark mode: run N collect+draw cycles without terminal init
    const auto cycles = cli.benchmark_cycles.value();

    // Initialize shared state (config, theme, term dimensions)
    // This is needed because collectors and draw functions read config and theme
    // Set Term dimensions to reasonable defaults since we don't have a real terminal
    Term::width = 200;
    Term::height = 50;

    // Load config and theme (needed by collectors and draw functions)
    // ... use the same config loading path as normal startup, but skip terminal init

    // Initialize Shared (loads initial collector data)
    Shared::init();

    // Timing vectors
    struct CycleTiming {
        double wall_us;       // Total cycle wall time in microseconds
        double collect_us;    // Total collect time
        double draw_us;       // Total draw time
    };
    std::vector<CycleTiming> timings;
    timings.reserve(cycles);

    for (uint32_t i = 0; i < cycles; i++) {
        auto cycle_start = std::chrono::high_resolution_clock::now();

        // Collect phase
        auto collect_start = std::chrono::high_resolution_clock::now();
        auto& cpu = Cpu::collect();
        auto& mem = Mem::collect();
        auto& net = Net::collect();
        auto& proc = Proc::collect();
        auto collect_end = std::chrono::high_resolution_clock::now();

        // Draw phase (output discarded -- we only measure time)
        auto draw_start = std::chrono::high_resolution_clock::now();
        auto cpu_out = Cpu::draw(cpu, /* ... appropriate args ... */);
        auto mem_out = Mem::draw(mem, /* ... appropriate args ... */);
        auto net_out = Net::draw(net, /* ... appropriate args ... */);
        auto proc_out = Proc::draw(proc, /* ... appropriate args ... */);
        auto draw_end = std::chrono::high_resolution_clock::now();

        auto cycle_end = std::chrono::high_resolution_clock::now();

        timings.push_back({
            std::chrono::duration<double, std::micro>(cycle_end - cycle_start).count(),
            std::chrono::duration<double, std::micro>(collect_end - collect_start).count(),
            std::chrono::duration<double, std::micro>(draw_end - draw_start).count()
        });
    }

    // Output JSON
    // Use fmt::format to construct JSON (btop already depends on fmt)
    // Format: { "cycles": N, "timings": [ { "wall_us": ..., "collect_us": ..., "draw_us": ... }, ... ], "summary": { "mean_wall_us": ..., "mean_collect_us": ..., "mean_draw_us": ... } }

    // Calculate summary statistics
    double sum_wall = 0, sum_collect = 0, sum_draw = 0;
    for (const auto& t : timings) {
        sum_wall += t.wall_us;
        sum_collect += t.collect_us;
        sum_draw += t.draw_us;
    }

    // Print JSON to stdout
    // ... construct and print JSON with fmt::format

    return 0;
}
```

**IMPORTANT implementation details:**

a) **Study the existing btop_main initialization sequence carefully.** The normal path does: CLI parse -> locale setup -> config load -> theme load -> term init -> signal setup -> Runner thread. Benchmark mode needs locale + config + theme + Shared::init, but NOT term init or signal handlers.

b) **Draw function signatures:** Look at how Runner calls draw functions in btop.cpp. The draw functions take boolean flags (like `force_redraw`, `data_same`). For benchmarking, use `true` for force_redraw and `false` for data_same to simulate worst-case draw.

c) **GPU support:** If GPU collectors are present (BTOP_GPU defined), include them in the benchmark cycle or skip with a comment explaining why.

d) **Error handling:** Wrap the benchmark loop in try/catch. If a collector crashes, print an error JSON and exit with non-zero status.

e) **JSON output:** Use raw fmt::format to build JSON. Do NOT add a JSON library dependency. The output format must be simple enough for a jq one-liner to parse.

f) **Output format specification:**
```json
{
  "benchmark": {

      "mean_collect_us": 30000,

      "min_wall_us": 40000,

    }
  }
}
```
