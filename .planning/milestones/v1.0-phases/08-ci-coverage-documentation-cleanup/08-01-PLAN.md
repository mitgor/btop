  <action>
Create a NEW workflow file `.github/workflows/pgo-validate.yml` with a scheduled PGO build validation job.

**Why a separate workflow (not in cmake-linux.yml):** cmake-linux.yml has `paths:` filters under `push:` and `pull_request:`. Adding a `schedule:` trigger would cause ALL jobs in cmake-linux.yml to run (build matrix + sanitizers) on schedule, which is wasteful. A separate workflow contains only the PGO job.

The workflow should have:
- `name: PGO Build Validation`
- Triggers: `schedule` (weekly Monday 6 AM UTC: `'0 6 * * 1'`) + `workflow_dispatch` for manual triggering
- `permissions: contents: read` (read-only, no push needed)
- Single job `pgo-validate` running on `ubuntu-24.04`

Job steps:
1. `actions/checkout@v6`
2. Install Clang 19: `sudo apt-get update && sudo apt-get install -y clang-19 llvm-19 ninja-build`
   (Ubuntu 24.04 has clang-19 in repos, no need for llvm.sh)
3. Run PGO build script:
   ```yaml
   - name: Run PGO build
     env:
       CXX: clang++-19
     run: ./scripts/pgo-build.sh Release
   ```
4. Verify the PGO binary exists and runs:
   ```yaml
   - name: Verify PGO binary
     run: |
       test -f build-pgo-use/btop
       ./build-pgo-use/btop --benchmark 5
       echo "PGO binary validated successfully"
   ```

Do NOT add `llvm.sh` script download -- Ubuntu 24.04 repos include clang-19 natively. Use `clang++-19` as the compiler (with version suffix) since that is the package-installed binary name.

The `pgo-build.sh` script already handles Clang detection, profraw merging, error checking, and the full 4-step PGO workflow. The CI job just needs to provide Clang and invoke the script.
