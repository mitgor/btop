  <name>Task 1: Fix Ctrl+C hang — clean_quit re-entrancy guard and main loop exit</name>

  <action>
Two changes to fix the Ctrl+C hang:

**Change A — clean_quit() re-entrancy guard (line 216):**

Replace the current guard:
```cpp
if (Global::app_state.load() == AppStateTag::Quitting) return;
Global::app_state.store(AppStateTag::Quitting);
```

With a separate static bool re-entrancy flag:
```cpp
static bool clean_quit_entered = false;
if (clean_quit_entered) return;
clean_quit_entered = true;
Global::app_state.store(AppStateTag::Quitting);
```

Why: transition_to() stores AppStateTag::Quitting to the shadow atomic at line 1010 BEFORE calling on_enter(Quitting) at line 1017. When on_enter(Quitting) calls clean_quit(), the old guard sees Quitting already set and returns early, never actually quitting. A separate static bool is not set by transition_to, so clean_quit proceeds correctly. The static bool also correctly prevents genuine re-entrancy (e.g., signal during cleanup).

**Change B — main loop exit after Quitting/Error (after the drain loop, around line 1454):**

After the drain loop's while block (line 1445-1454), add an outer loop exit check:
```cpp
{
    auto tag = to_tag(app_var);
    if (tag == AppStateTag::Quitting or tag == AppStateTag::Error) break;
}
```

Place this immediately after the drain loop's closing brace and before the "//? 2. Handle state chains" comment. This ensures the outer while(true) loop exits when the app enters a terminal state. The drain loop's break at line 1453 only breaks out of the inner while, not the outer.

Note: In practice, on_enter(Quitting) calls clean_quit() which calls _Exit()/_quick_exit(), so the process terminates inside the drain loop. This outer break is a defense-in-depth safety net for any edge case where clean_quit might return or for Error states.

  <name>Task 2: Fix resume-no-redraw — add on_event(Sleeping, Resume) handler</name>
  <files>src/btop.cpp</files>
  <action>
Add a new on_event overload for the (Sleeping, Resume) pair. Place it BEFORE the catch-all on_event(auto, auto) at line 333 (order matters for overload resolution):

```cpp
static AppStateVar on_event(const state::Sleeping&, const event::Resume&) {
	return state::Resizing{};  // Route through on_enter(Resizing) for full redraw
}
```

Why: When btop resumes from Ctrl+Z (SIGCONT), the state is Sleeping and a Resume event is dispatched. Without this handler, the catch-all returns Sleeping unchanged, and section 2 transitions Sleeping->Running directly via on_enter(Running) which does nothing. By returning Resizing{}, the transition fires on_enter(Resizing) which calls term_resize(true), Draw::calcSizes(), Runner::screen_buffer.resize(), and Runner::run("all", true, true) — providing the full redraw the user expects.

Place the new overload after the existing on_event(Running, ThreadError) at line 331 and before the catch-all at line 333. This ensures the compiler selects the specific overload over the catch-all template.

Do NOT modify the existing Sleeping block in section 2 (lines 1468-1474). It serves as a safety fallback if somehow no Resume event is queued.
