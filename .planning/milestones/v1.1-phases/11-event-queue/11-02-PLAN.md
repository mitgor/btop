**Step 1: Replace _signal_handler body (lines ~294-330)**

Replace the current signal handler with the event-pushing version. The new handler is trivially async-signal-safe:

```cpp
static void _signal_handler(const int sig) {
    switch (sig) {
        case SIGINT:
            Global::event_queue.push(event::Quit{0});
            break;
        case SIGTSTP:
            Global::event_queue.push(event::Sleep{});
            break;
        case SIGCONT:
            Global::event_queue.push(event::Resume{});
            break;
        case SIGWINCH:
            Global::event_queue.push(event::Resize{});
            break;
        case SIGUSR1:
            break;  // Input::poll interrupt — no event needed
        case SIGUSR2:
            Global::event_queue.push(event::Reload{});
            break;
    }
    // Wake main loop from pselect() — kill() is async-signal-safe
    if (sig != SIGUSR1) {
        Input::interrupt();
    }
}
```

Key changes from old handler:
- SIGINT: No more `if (Runner::active)` branch. Just push `Quit{0}`. The main loop handles the active/inactive distinction.
- SIGTSTP: No more `if (Runner::active)` branch. Just push `Sleep{}`.
- SIGCONT: No more calling `_resume()` directly. Push `Resume{}`.
- SIGWINCH: No more calling `term_resize()` directly. Push `Resize{}`.
- SIGUSR1: Unchanged (no-op, used for pselect interrupt).
- SIGUSR2: No more `app_state.store(Reloading)`. Push `Reload{}`.
- ALL handlers (except SIGUSR1): Call `Input::interrupt()` to wake main loop.

**Step 2: Add process_signal_event overloads before main()**

Add overloaded `process_signal_event` functions above `main()` but below the existing helper functions (`_sleep`, `_resume`, `term_resize`, `clean_quit`, etc.). These translate events back to the same behavior the old signal handlers produced:

```cpp
//* Event processors — translate signal events to state changes and actions.
//* Called from the main thread (safe to call any function).
static void process_signal_event(const event::Quit& e) {
    if (Runner::active) {
        Global::app_state.store(AppState::Quitting);
        Runner::stopping = true;
    } else {
        clean_quit(e.exit_code);
    }
}

static void process_signal_event(const event::Sleep&) {
    if (Runner::active) {
        Global::app_state.store(AppState::Sleeping);
        Runner::stopping = true;
    } else {
        _sleep();
    }
}

static void process_signal_event(const event::Resume&) {
    _resume();
}

static void process_signal_event(const event::Resize&) {
    term_resize();
}

static void process_signal_event(const event::Reload&) {
    Global::app_state.store(AppState::Reloading);
}

// No-ops for event types not used in Phase 11 (included in variant for Phase 12):
static void process_signal_event(const event::TimerTick&) {}
static void process_signal_event(const event::ThreadError&) {}
```

**IMPORTANT behavioral equivalence details:**
- `process_signal_event(Quit)` preserves the Runner::active check — if runner is active, set state + stop runner; otherwise call clean_quit directly. This matches the old SIGINT handler exactly.
- `process_signal_event(Sleep)` preserves the same Runner::active check pattern.
- `process_signal_event(Resume)` calls `_resume()` unconditionally (same as old SIGCONT handler).
- `process_signal_event(Resize)` calls `term_resize()` without arguments (same as old SIGWINCH handler). Note: this is now called from the main thread, which is SAFER than before (the old code called it from signal context, which was a pre-existing signal-safety bug).
- `process_signal_event(Reload)` sets the state to Reloading. Note: The old handler also called `Input::interrupt()` after setting state, but now `Input::interrupt()` is already called in the signal handler after pushing the event, so this is covered.

**Step 3: Add event drain loop at top of main loop (before the state check)**

In the main loop (line ~1309, right after `while (not true not_eq not false) {`), add the event drain before the existing `const auto state = Global::app_state.load(...)` line:

```cpp
//? Drain signal events from the queue (pushed by signal handlers)
while (auto ev = Global::event_queue.try_pop()) {
    std::visit([](const auto& e) { process_signal_event(e); }, *ev);
    // Early exit on terminal states to preserve priority semantics
    auto s = Global::app_state.load(std::memory_order_acquire);
    if (s == AppState::Quitting or s == AppState::Error) break;
}
```

This drain loop runs at the top of every main loop iteration, BEFORE the existing if/else if state check chain. Events are processed FIFO, but the early-exit ensures that once a terminal state (Quitting/Error) is reached, further events are ignored (preserving the priority semantics documented in btop_state.hpp).
