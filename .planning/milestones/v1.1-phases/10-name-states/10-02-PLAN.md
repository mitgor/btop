if (state == AppState::Error) {
    clean_quit(1);

    clean_quit(0);
}
else if (state == AppState::Sleeping) {
    auto expected = AppState::Sleeping;
    Global::app_state.compare_exchange_strong(expected, AppState::Running);
    _sleep();

    init_config(cli.low_color, cli.filter);

IMPORTANT for the resize check: The original code called `term_resize(Global::resized)` which also might SET `Global::resized = true` internally. Then the next check `if (Global::resized)` catches both the case where it was already Resizing AND the case where term_resize just set it. So we need to re-check `Global::app_state.load()` after calling `term_resize()`. The `or` condition handles both cases.

Line ~1278: Replace `Global::resized = false;` with:
```cpp

```

Line ~1355: Replace `not Global::resized` with `Global::app_state.load() != AppState::Resizing`

CRITICAL CONSTRAINTS:
- Use `compare_exchange_strong` when clearing state back to Running. This prevents swallowing a higher-priority state set by a signal handler during the transition (Pitfall 3 from research).
- Set `exit_error_msg` BEFORE `app_state.store(AppState::Error)` and use `memory_order_release` for the store. The main loop loads with `memory_order_acquire`. This guarantees the error message is visible when the main loop sees the Error state (Pitfall 5 from research).
- Do NOT use a switch statement in the main loop -- keep the if/else if structure because the flow is not a simple dispatch (term_resize is called unconditionally between the priority checks and the resize handling).
- Do NOT touch Runner's own atomic&lt;bool&gt; flags (active, stopping, waiting, redraw) -- those belong to Phase 14.
